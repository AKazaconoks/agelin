## The Actor Model

The two codes map onto two distinct gatekeeping questions:

| Code | Question | Remedy |
|------|----------|--------|
| **401 Unauthorized** | "**Who are you?**" (authentication) | Provide or re-provide credentials |
| **403 Forbidden** | "**Are you allowed?**" (authorization) | Re-authenticating won't help |

The name "Unauthorized" in 401 is a longstanding **misnomer** — the spec's normative text makes clear it's about *authentication*, not authorization.

---

## What RFC 9110 Says

**401** — The server lacks valid authentication credentials for the target resource. The server **MUST** send a `WWW-Authenticate` header telling the client *how* to authenticate. The implication is always: "Come back with valid proof of who you are."

**403** — The server understood the request but refuses to fulfill it. Unlike 401, the server is *not* asking the client to re-authenticate. RFC 9110 explicitly states: "If the client's credentials are already known to the server and are insufficient, sending new credentials would not change the outcome."

---

## Your Two Scenarios

- **User is not logged in (anonymous request):** Return **401**. The server doesn't know who they are; provide a `WWW-Authenticate` challenge.
- **User is logged in but lacks the required role/group:** Return **403**. Authentication succeeded; the problem is insufficient privilege. Returning 401 here is semantically wrong — it implies "re-authenticate," which is the wrong advice.

A useful analogy: a security guard at a government building.
- **401** = "Show me your ID."
- **403** = "I see your ID. You're not cleared for this floor."

---

## Other Codes Used in Practice

| Code | When |
|------|------|
| **302** (redirect to `/login`) | Common browser convention for the "not logged in" case. Not pure HTTP semantics, but practical for HTML flows. APIs should still use 401/403. |
| **404** | RFC 9110 explicitly sanctions using 404 when even the *existence* of the resource is sensitive — security-through-obscurity to prevent information leakage. |

---

**The single rule to remember:** 401 always means "authenticate (or re-authenticate)"; 403 means "authentication is not the problem." Any system returning 401 for a logged-in user who simply lacks a role is violating the spec.
